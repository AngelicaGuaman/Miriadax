Pruebas de Aceptación

¿Qué son y para qué sirven?

Las pruebas de aceptación verifican que el sistema que recibe funciona y lo hace de acuerdo con las especificaciones. Las pruebas de aceptación se realizan de acuerdo con protocolos específicos del producto en nuestra fábrica o en terreno.

Para automatizar este tipo de pruebas se ha optado por la herramienta FitNesse. FitNesse es un servidor web de código abierto, se trata de una herramienta para la especificación y verificación de los criterios de aceptación del software. Esta herramienta está basada en Framework for Integrated Test <FIT>, para automatizar pruebas y mejorar la colaboración en el desarrollo entre clientes, programadores y tester. Fit permite crear tablas que contienen información relacionada con las pruebas, haciéndolo sencillo de entender por los usuarios. Inicialmente sólo se podía trabajar con FIT, aunque posteriormente apareció una segunda alternativa llamada Slim. El siguiente diagrama muestra como FitNesse trabaja con los dos sistemas de pruebas.

El funcionamiento de FitNesse es simple, se crea una Wiki que contiene un conjunto de pruebas, para escribir estas pruebas se utilizan tablas en las que se le dice a la aplicación bajo test las acciones a realizar, así como los resultados que se esperan obtener.
